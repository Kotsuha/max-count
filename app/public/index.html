max count


max count: 理論値数100を超えている人をライバルしてるだけのIDです。

How to use

Enter your ID, press the submit button and the check will start.

The check may take a fairly long time so it is recommended to just leave it. Do not close or refresh the page if you would like to view logs or be able to cancel the request.

After the check is done, if there are enough MAXs found, the ID will be added to max count's rival list automatically.

Please note that:

  • ID format: Please enter the ID shown in the URL (the number after playerid=) instead of the one shown in the My Page (マイページ) table. For example, if your ID is 1, please enter 1 rather than 000001.
  • Maximum number of tasks: There is a maximum number of IDs this tool can check at the same time. Please refer to the tab title. When the maximum number is reached, newer requests are not accepted and you'll have to wait. This is in order to not make the LR2IR server too busy. The number is rather small because I don't think there will be many people using this...
  • Canceling a request: A request can be canceled by only the same tab which sends it. A request cannot be canceled after the tab is closed or refreshed.

使い方

IDを入力し、送信ボタンを押してください。そうするとチェックが始まります。

理論値数が100を超える場合、チェック完了後そのIDが自動的にmax countのライバルリストに追加されます。

チェックにはかなり長い時間がかかりますので、チェックする期間、そのまま放置することをお勧めします。もしログを表示したい、またはリクエストを取り消したいなら、ページを閉じたり更新したりしないでください。

注意：

  • IDのフォーマット：マイページの表に表示されているIDではなく、URLに表示されているID（playerid=の後の数字）を入力してください。たとえば、IDが1の場合は、000001ではなく1を入力してください。
  • 人数の上限：同時に使用できる最大人数があります。タブのタイトルを参照してください。LR2IRサーバーをビジー状態にしないためです。そして、私はこれを使用している多くの人々が存在するとは思わない...
  • タスク数の上限：このツールには、同時にチェックできるIDの最大数（すべてのユーザーが共有）があります。タブのタイトルを参照してください。LR2IRサーバーをビジー状態にしないためです。そして、私はこれを使用している多くの人々が存在するとは思わない...
  • リクエストを取り消す：元々リクエストを送信したタブでそのリクエストを取り消せます。タブが閉じられて、もしくは更新されるなら、そのリクエストが取り消されないようになります。

用法

輸入你的 ID 並按下提交按鈕，檢查就會開始了。

檢查可能會花上滿長一段時間，建議放置，並且途中不要關閉或重整網頁，否則將無法看 log 或取消請求。

檢查結束後，如果有找到足夠的理論值，該 ID 應該就會被加進 max count 的ライバル了。

注意：

  • ID 的格式：ID 請用網址 playerid= 後面的數字，不要用マイページ表格裡的。例如：如果你的 ID 是 1，請輸入 1，不要輸入 000001。
  • 任務數量上限：為了不要讓 LR2IR 伺服器變的很忙，並且我覺得會來用這個的人應該不多吧... 總之有能夠同時檢查的 ID 數量上限 (所有使用者共享)，請參照分頁標題。超越上限的請求將不被受理，需等其他 ID 被檢查完。
  • 取消請求：哪個分頁送出的請求只能由哪個分頁取消。重整或關閉分頁以後，請求就無法取消。
